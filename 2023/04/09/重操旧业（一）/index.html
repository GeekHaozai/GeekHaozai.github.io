C#和.NET框架（.NET Framwork）

1.Win32 API

Win32 API 是由 Microsoft 使用 C 和汇编语言编写的，这些函数和接口是以 C 语言风格的函数接口形式提供的。在 C 语言中，使用 extern “C” 来声明 C 语言函数接口，以便于其它编程语言可以通过其名称进行调用。这种形式的函数接口也被称为 C 语言调用约定（Calling Convention），Win32 API 中使用的是标准的 C 语言调用约定。

使用 Win32 API 就是调用 Windows 操作系统中的 DLL 文件来使用其中的函数和接口。这些 DLL 文件中包含了大量的 Win32 API 函数和接口，应用程序可以通过加载和调用这些 DLL 文件来使用 Win32 API。不过需要注意的是，Win32 API 中的函数和接口有着严格的参数和返回值要求，使用时需要按照其规范进行调用，否则可能会导致应用程序出现错误或异常。

2.MFC（Microsoft Foundation Classes，Microsoft 基础类库）

MFC 则是在 Win32 API 基础上，使用 C++ 封装了一些常用的 GUI 控件和应用程序框架，提供了一些面向对象的开发接口，使得开发者可以更加方便地使用 Win32 API 来开发 Windows 应用程序。使用 MFC 开发程序时，开发者可以通过封装好的 C++ 类来创建窗口、处理消息、操作文件等等，而无需直接调用 Win32 API。MFC 本质上是对 Win32 API 的一种封装和扩展，因此可以看作是 Win32 API 的一种高级封装。

但是只能在 Visual C++ 环境下使用。

3.COM（Component Object Model）

COM 组件通常是通过 DLL 文件实现的。COM 组件包含一个或多个对象，每个对象都有一个标识符和一组方法。在运行时，应用程序可以通过调用 COM 组件的接口，与其中的对象进行交互，并使用其提供的方法来实现特定的功能。COM 组件的接口是通过虚函数表（VTable）来实现的，这个虚函数表存储了 COM 组件的方法地址和其他相关信息。当应用程序调用 COM 组件的方法时，它实际上是在调用 COM 组件的函数指针，然后由 DLL 文件负责实际执行这些方法。因此，COM 组件的 DLL 文件是实现 COM 组件接口的关键。

(上面这段话暂时还不是很懂)

比对

COM、MFC 和 Win32 API 都是 Windows 平台下的应用程序开发技术，但它们各自的定位和作用略有不同。

COM 是一种组件对象模型，用于实现组件之间的互操作性和代码重用，它提供了一种标准的接口和协议，使得不同的软件组件可以通过相同的接口进行交互，从而实现更加灵活和可扩展的应用程序设计。

MFC（Microsoft Foundation Class）是一个基于 C++ 的类库，它封装了 Win32 API 的许多功能，提供了更加简单易用的编程接口。MFC 主要用于 Windows 平台下的 GUI 应用程序开发，它提供了许多 UI 控件和窗口管理功能，帮助开发人员快速构建 Windows 应用程序。

Win32 API 是 Windows 操作系统提供的一组函数和接口，它们是操作系统和应用程序之间的接口，可以访问和控制操作系统的各种功能和资源，例如文件、进程、窗口、消息等。Win32 API 的主要作用是提供底层系统编程接口，帮助开发人员编写更加高效和精密的应用程序。

总的来说，COM 是一种组件技术，MFC 是一种应用程序框架，Win32 API 是一种系统编程接口。它们之间有一些交叉和重叠，但各自的定位和作用还是有所不同的。

纯Win32 API不是面向对象的；MFC虽然面向对象但是它并不一致，变得陈旧；COM实际代码复杂，丑陋、不雅！

.NET Framwork

主要组成部分：

  1. CLR（Common Language Runtime）：.NET Framework 的核心组件之一，负责解释和执行 .NET 应用程序的 IL（Intermediate Language）代码，并提供垃圾回收、异常处理、安全性、线程管理等功能。
  2. FCL（Framework Class Library）：.NET Framework 的类库，提供了许多用于开发 .NET 应用程序的基础类、集合类、IO 类、数据库访问类、网络类、安全类、XML 类、Web 类等。
  3. ADO.NET：.NET Framework 中用于访问数据库的组件，包括 DataSet、DataAdapter、Connection 等，可以连接多种不同的数据源，如 SQL Server、Oracle、MySQL 等。
  4. ASP.NET：.NET Framework 中用于开发 Web 应用程序的组件，包括 Web Forms、MVC、Web API 等，提供了一系列易于使用的功能和工具，支持多种 Web 开发模型。
  5. Windows Forms：.NET Framework 中用于开发桌面应用程序的组件，提供了一套基于 Windows 的用户界面，支持多种控件和布局方式，可以快速创建美观且易于使用的 Windows 应用程序。
  6. WPF（Windows Presentation Foundation）：.NET Framework 中用于开发 Windows 应用程序的 UI 框架，提供了一套矢量图形、动画、多媒体等功能，支持 XAML 标记语言，可以创建高度可定制的用户界面。
  7. WinRT（Windows Runtime）：.NET Framework 中用于开发 Windows 应用程序的新型 API，支持多种编程语言，如 C++、C#、VB.NET 等，提供了一套现代化的开发体验和功能。

具体说说CLR、FCL、BCL、CIL、CLI

CLR

CLR 负责将代码编译成中间语言（Intermediate Language，简称 IL），并在运行时将其转换为本机代码执行。

由于 CLR 提供了一个统一的执行环境，所以不同的 .NET 语言可以通过 CLR 进行交互和协作，比如 C#、VB.NET、F# 等。此外，CLR 还支持在托管代码[^什么是托管代码]中调用非托管代码，比如 C++ DLL 等，从而实现对现有代码的重用和集成。

FCL、BCL

BCL（Basic Class Library）是基类库，FCL（Framwork Class Library）是框架类库，严格来说BCL是FCL的子集，但有时候不加区别。

CIL

IL（Intermediate Language）和CIL（Common Intermediate Language）是同一个概念。IL或者CIL都是指.NET Framework中使用的中间语言，它是一种面向对象的、类型安全的、与平台无关的汇编语言，用于描述.NET Framework中的程序代码。

.NET Framework中的编译器将源代码编译为IL代码，并将其保存在扩展名为.dll（动态链接库）或.exe（可执行文件）的文件中。当.NET应用程序运行时，CLR将IL代码编译为本机代码，然后才能够执行。

在.NET Framework 1.x版本中，中间语言称为IL；而在.NET Framework 2.0版本之后，中间语言正式改名为CIL。因此，IL和CIL的本质是一样的，只是命名上的区别。

编译成CIL之后程序集中的可执行代码只会在需要的时候被JIT（just-in-time）编译器编译为本机代码[^什么是本机代码]

CLI

CLI是Common Language Infrastructure的缩写，中文名为公共语言基础设施。它是由国际标准化组织ISO和国际电工委员会IEC制定的，用于定义和实现跨平台、跨语言的程序集的一套规范和标准。

虽然不需要了解CLI的具体规范细节，但是至少应该熟悉公共类型系统和公共语言规范的含义和用途

CLI的具体介绍

[^什么是托管代码]:托管代码是指在.NET Framework下运行的代码，它在运行时受到.NET Framework的管理和控制。在.NET Framework中，托管代码是由公共语言运行时（CLR）执行的，CLR提供了一系列服务，例如自动垃圾回收、代码安全性、异常处理等，以保证代码的运行环境安全和可靠。相对于非托管代码，托管代码更加安全，易于维护和管理。
[^什么是本机代码]:本机代码（Native code）是与特定硬件和操作系统平台相关的机器码指令，它是由CPU直接执行的，因此能够提供最高的性能和最好的可执行性。与IL代码相比，本机代码不需要CLR进行中间语言到本机代码的转换，因此运行速度更快。在.NET Framework中，当CLR将IL代码编译成本机代码后，它会被缓存，这样当程序下一次执行时，不需要再次将IL代码编译成本机代码，提高了应用程序的性能。

1.公共类型系统

CTS（Common Type System），定义了那些在托管代码中一定会使用的类型的特征

  • CTS定义了一组丰富的内置类型，以及每种类型独有的、固有的特性
  • .NET兼容编程语言提供的类型通常映射到CTS中已定义的内置类型集的某一个特殊子集
  • CTS的一个重要特征就是所有类型都继承自公共的基类——object
  • 使用CTS可以确保系统类型和用户定义类型能够被任何.NET兼容的语言所使用

2.公共语言规范

CLS（Common Lauguage Specification）详细地说明了一个.NET兼容编程语言的规则、属性和行为，其主题包括数据类型、类结构和参数传递

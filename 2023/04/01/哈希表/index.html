各种语言中的哈希表

  • 哈希表（Hash Table），也称为散列表，是一种数据结构，它可以将键（key）和值（value）之间建立一种映射关系。哈希表使用哈希函数将键转换为对应的数组索引，然后将值存储在该索引位置的数组元素中。哈希函数是一种将任意长度的输入转换为固定长度的输出的算法，可以将键映射到一个固定的位置，从而使得查找、插入和删除操作的时间复杂度可以达到O(1)。

  • 哈希表通常是由一个固定大小的数组和一个哈希函数组成。数组的大小通常是根据哈希表需要存储的键值对的数量来确定的，而哈希函数的设计也需要考虑键的分布情况，以尽可能避免冲突，提高哈希表的性能。

  • 哈希函数（Hash Function）是一种将任意大小的数据（如字符串、文件或整数）映射为固定大小的数据（通常是一个较短的固定长度的哈希值或哈希码）的函数。哈希函数的主要作用是将任意长度的数据映射为一个较短的固定长度的哈希值，从而方便进行数据存储和快速的数据查找。

    哈希函数具有以下特点：

    1. 输入数据相同，哈希值也应该相同；
    2. 哈希函数应该快速计算；
    3. 不同的输入数据可能会产生相同的哈希值，这种情况称为哈希冲突；
    4. 好的哈希函数应该尽可能地减少哈希冲突的概率，以提高哈希表的效率；
    5. 哈希函数应该满足均匀性，即输入数据的微小变化应该导致哈希值的大幅变化，从而减少哈希冲突的概率。

C#

在C#中，哈希表可以使用System.Collections命名空间下的Hashtable类来实现。Hashtable类提供了一系列方法来对键值对进行添加、删除、查找等操作。

1.Hashtable实现

下面是一个使用C#中的Hashtable类来实现简单哈希表的示例代码：

using System;
using System.Collections;

class Program {
    static void Main(string[] args) {
        // 创建一个Hashtable对象
        Hashtable hash = new Hashtable();

        // 向哈希表中添加键值对
        hash.Add("key1", "value1");
        hash.Add("key2", "value2");
        hash.Add("key3", "value3");

        // 通过键来查找值
        Console.WriteLine("value of key1: " + hash["key1"]);

        // 删除键值对
        hash.Remove("key2");

        // 遍历哈希表中的键值对
        foreach (DictionaryEntry entry in hash) {
            Console.WriteLine(entry.Key + ": " + entry.Value);
        }
    }
}

注：

DictionaryEntry结构体是在System.Collections命名空间下定义的。它是一个通用的键值对结构体，用于表示一个字典中的一个键值对

2.Dictionary<TKey, TValue>实现

下面是一个使用System.Collections.Generic命名空间下的Dictionary<TKey, TValue>泛型类的例子：

using System;
using System.Collections.Generic;

class Program
{
    static void Main()
    {
        // 创建一个Dictionary对象
        var dict = new Dictionary();

        // 向Dictionary中添加一些键值对
        dict.Add("apple", 1.99);
        dict.Add("banana", 0.99);
        dict.Add("orange", 1.49);

        // 遍历Dictionary中的所有键值对
        foreach (var entry in dict)
        {
            Console.WriteLine("Key: {0}, Value: {1}", entry.Key, entry.Value);
        }
    }
}

两者的比较：

Dictionary<string, double>和Hashtable都是C#中用于存储键值对的数据结构，它们的区别主要在于：

  1. 类型安全：Dictionary<TKey, TValue>是一个泛型类，可以在编译时就指定键和值的具体类型，从而避免了类型转换错误；而Hashtable是一个非泛型的集合类，它存储的键和值都是object类型，需要在运行时进行类型转换，可能会导致性能损失和类型安全问题。
  2. 性能：由于Dictionary<TKey, TValue>是一个泛型类，它的内部实现可以针对具体的键和值类型进行优化，从而提高性能。而Hashtable是一个通用的集合类，它的内部实现相对比较通用，可能会存在一些性能上的损失。
  3. 功能：Dictionary<TKey, TValue>提供了一些Hashtable不具备的功能，例如TryGetValue方法可以在字典中查找指定键的值，而Hashtable需要使用ContainsKey和索引访问的方式进行查找。此外，Dictionary<TKey, TValue>还提供了一些LINQ相关的扩展方法，使得对字典的操作更加方便。

因此，如果在C#中需要存储键值对，推荐使用Dictionary<TKey, TValue>来代替Hashtable，以获得更好的类型安全、性能和功能。

C++

注：

C++中的哈希表是由unordered_map类实现的，它使用了开链法（chaining）来处理冲突。C++11标准引入了新的哈希表实现方式，即unordered_map类和unordered_set类，它们使用了桶（bucket）来处理冲突，提高了哈希表的性能。

1.unordered_map类

下面是一个使用C++标准库中的unordered_map类来实现简单哈希表的示例代码：

#include 
#include 

int main() {
    // 创建一个unordered_map对象
    std::unordered_map map;

    // 向哈希表中添加键值对
    map["key1"] = "value1";
    map["key2"] = "value2";
    map["key3"] = "value3";

    // 通过键来查找值
    std::cout << "value of key1: " << map["key1"] << std::endl;

    // 删除键值对
    map.erase("key2");

    // 遍历哈希表中的键值对
    for (const auto& kv : map) {
        std::cout << kv.first << ": " << kv.second << std::endl;
    }

    return 0;
}

int main()
{
    std::unordered_map word_count;

    // 读入一行文本
    std::string line;
    std::getline(std::cin, line);

    // 将每个单词加入哈希表中
    std::string word;
    for (auto ch : line) {
        if (std::isalpha(ch)) {
            word.push_back(std::tolower(ch));
        } else if (!word.empty()) {
            ++word_count[word];
            word.clear();
        }
    }
    if (!word.empty()) {
        ++word_count[word];
    }

    // 输出每个单词的出现次数
    for (const auto& kv : word_count) {
        std::cout << kv.first << ": " << kv.second << std::endl;
    }

    return 0;
}

拓展（std：：map和std::set)

std::map和std::set是C++ STL中的关联容器，它们使用红黑树实现。红黑树是一种自平衡二叉搜索树，可以保证在最坏情况下，每个操作的时间复杂度都是O(log n)。

与哈希表不同的是，std::map和std::set中的元素是按照键（或值）的顺序排列的，而不是按照哈希值的顺序排列的。这意味着，当需要对元素进行排序时，使用std::map和std::set会更加方便。

std::map和std::set本身不是基于哈希表实现的，但是C++ STL中也提供了基于哈希表实现的关联容器std::unordered_map和std::unordered_set。这些容器使用哈希函数来映射键（或元素）到桶中，并使用链表或者红黑树来解决哈希冲突。因此，std::unordered_map和std::unordered_set可以看作是基于哈希表实现的std::map和std::set。
